# Radar

This directory contains individual technology blips as Markdown files. Each blip represents a technology, tool, technique, platform, or framework that teams in the Portfolio & Delivery organization have experience with.

## Directory Structure

Organize blips by quadrant (category):

```
radar/
├── techniques/          # Development and process techniques
├── platforms/           # Infrastructure, deployment, and cloud platforms
├── tools/              # Development and operational tools
└── tips/               # Tips and recommendations for working with technology
```

## Ring Meanings

- **adopt**: Proven technologies we use with confidence in production
- **trial**: Worth pursuing in projects that can handle the risk
- **assess**: Worth exploring to understand potential
- **hold**: Proceed with caution or avoid based on team experience
