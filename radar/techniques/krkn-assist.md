Enables internal teams, community users and customers to get chaos commands
to run given a use case as input without having to learn the Krkn framework
and manually create configurations.

Simplifies chaos engineering by providing AI-generated chaos commands based
on natural language use case descriptions. Lowers the barrier to entry for
chaos testing.
