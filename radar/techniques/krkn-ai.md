Supercharging chaos testing using AI. Identifies and focuses on scenarios
that have high probability to cause disruption instead of running hundreds
of scenarios - saves money, engineering hours, and reduces fault space.
Adds new test cases for new features as product evolves.

Working with HCM SRE team on ADR for automated workflows where teams can
define their SLOs and run Krkn-AI tests. Continuing work on issues from
HCM SRE pilot testing and community engagement. Helps scale chaos test
coverage for tens of services/products.
